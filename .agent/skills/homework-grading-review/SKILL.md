# Homework Grading Review

Use this repo-local skill to regrade homework as an independent review pass, compare it with completed grading artifacts, and isolate high-risk score differences. Treat `work/rubric/rubric.json`, `work/answers/*.answers.json`, `work/grading/*.grading.json`, `work/reports/*.csv`, and original submissions under `hw/` as inputs. The `work/answers/{student_id}.answers.json` files are generated by the `homework-grading` skill's student extraction, question splitting, and answer-alignment workflow. Write review outputs only under `work/review/`, except when a broken answers JSON must be regenerated as a prerequisite for reliable review grading.

Primary outputs:

- `work/review/regrading/{student_id}.grading.json`: second-pass grading JSON only
- `work/review/score_discrepancies.csv`: students with large original-vs-review score differences; the first column must be `student_id`
- `work/review/problem/`: copies of the original submission files for students listed in `score_discrepancies.csv`

## Boundaries

Use the `homework-grading` skill as the grading policy source for the review pass: same rubric, same per-scoring-point grading rules, same validation expectations. The review pass must not produce the normal homework-grading extra artifacts.

Do not create these review-pass artifacts unless the user explicitly asks:

- `work/review/reports/`
- `work/review/final/`
- review-pass `summary.csv`
- review-pass `review_queue.csv`
- per-student Markdown reports
- per-student final PDFs

Do not use this skill to:

- regenerate the rubric
- assign final grades directly from raw PDFs or images
- build a Web UI, database, API orchestration layer, or LangGraph flow

Do not overwrite `as/`, `hw/`, `work/grading/`, or `work/reports/` unless the user explicitly asks to apply fixes to original files. Put corrected grading drafts under `work/review/corrected_grading/`. Put second-pass review grading under `work/review/regrading/`.

For detailed policy, read `references/review_policy.md`. For output schemas and LLM review prompts, read:

- `references/review_schema.json`
- `references/discrepancy_schema.json`
- `references/prompt_templates.md`

## Workflows

### Second-Pass Grading Difference Review

Use this as the default workflow when the user asks to review grading by grading again, regrade for review, compare with existing grading, or find students whose scores differ a lot.

1. Validate inputs:
   - `work/rubric/rubric.json` must exist.
   - `work/grading/*.grading.json` must contain the original grading.
   - `hw/` must contain original student submissions.
   - Prefer existing `work/answers/{student_id}.answers.json` as the student-answer source so the review pass does not create extraction files.
2. Check every `work/answers/{student_id}.answers.json` before using it for review grading:
   - Confirm the JSON conforms to `.agent/skills/homework-grading/references/answers_schema.json` and has one answer object per rubric `question_id`.
   - Treat the file as broken if every question has identical or near-identical `answer_text`, `source_snippet`, or source-page metadata; if each answer repeats the full submission; if required questions are missing; or if the file is obviously copied from the wrong student.
   - Broken answers JSON was produced by the `homework-grading` answer extraction/alignment stage, not by this review comparison stage.
   - Regenerate broken answers JSON before second-pass grading by rerunning the `homework-grading` extraction, `split_questions.py`, and answer-alignment workflow for the affected student submission.
   - Preserve the bad file under `work/review/invalid_answers/{student_id}.answers.json` before replacing `work/answers/{student_id}.answers.json`, or write the regenerated file under `work/review/answers/{student_id}.answers.json` and use that copy if the user has not approved modifying `work/answers/`.
3. Load and follow the `homework-grading` skill's rubric and grading rules, but write only second-pass grading JSON:
   - Output path: `work/review/regrading/{student_id}.grading.json`
   - Do not run report, final PDF, summary, or review queue rendering commands.
   - Do not overwrite `work/grading/{student_id}.grading.json`.
4. Extract `student_id` from original submission filenames by the LMS-style numeric ID. Example:
   - filename: `Homework 1_2022233296_尝试_2026-04-18-17-58-06_SI251_HW1.pdf`
   - `student_id`: `2022233296`
5. Compare original and second-pass grading with a concrete quantitative rule. A student is a large-difference case if any condition holds:
   - absolute total-score difference is at least 2 points
   - total-score difference is at least 5% of total possible score
   - any single-question score difference is at least 1 point
   - any single-question score difference is at least 20% of that question's max score
   - either pass is missing a numeric total or comparable numeric question score
6. Put only large-difference students into `work/review/score_discrepancies.csv`.
   - The first CSV column must be `student_id`.
   - Include original total, review total, absolute difference, percent difference, changed questions, quantitative flags, and a concise reason based on question-level score and deduction-reason differences.
7. Copy the original homework files for every large-difference student into `work/review/problem/`.
   - Copy files from `hw/` whose filename contains the extracted `student_id`.
   - Preserve original filenames when possible.
   - If a student submitted multiple image/PDF files with the same ID, copy all of them.

Comparison command after second-pass JSON files exist:

```bash
python .agent/skills/homework-grading-review/scripts/compare_regrading_scores.py --root .
```

Useful threshold override example:

```bash
python .agent/skills/homework-grading-review/scripts/compare_regrading_scores.py --root . --total-points-threshold 3 --total-ratio-threshold 0.08
```

### Existing Artifact Audit

1. Validate inputs and repo paths.
2. Audit every `work/grading/*.grading.json` against the rubric.
3. Compare `summary.csv` and `grades.csv` with grading JSON.
4. Rebuild the review queue from discrepancies and uncertainty signals.
5. Render `work/review/review_report.md`.
6. Optionally render `work/review/review_report.pdf` from the Markdown report.

Commands:

```bash
python .agent/skills/homework-grading-review/scripts/validate_review_inputs.py --root .
python .agent/skills/homework-grading-review/scripts/audit_grading_json.py --root .
python .agent/skills/homework-grading-review/scripts/audit_summary_consistency.py --root .
python .agent/skills/homework-grading-review/scripts/build_review_queue.py --root .
python .agent/skills/homework-grading-review/scripts/render_review_report.py --root .
python .agent/skills/homework-grading-review/scripts/render_review_pdf.py --root .
```

Use `--student-id` to focus the JSON audit:


## Second-Pass Grading Rules

- Reuse `work/rubric/rubric.json`; do not regenerate it.
- Grade independently from the original score, but do not hide the fact that this is a review pass.
- Apply the same scoring-point rules as `homework-grading`.
- Use existing `work/answers/{student_id}.answers.json` only after the answer JSON health check passes. If it is missing, obviously incomplete, duplicated across questions, or otherwise unreliable, regenerate it with the `homework-grading` answer extraction/alignment workflow before grading the review pass.
- Validate each `work/review/regrading/{student_id}.grading.json` against the homework-grading grading schema when possible:

```bash
python .agent/skills/homework-grading/scripts/validate_grading.py --grading work/review/regrading/{student_id}.grading.json --rubric work/rubric/rubric.json --schema .agent/skills/homework-grading/references/grading_schema.json
```

- If validation fails, repair the review grading JSON and rerun validation.
- Never update original totals from the review pass automatically. `score_discrepancies.csv` is a review queue for TA judgment, not a final grade override.

## Large-Difference CSV Schema

`work/review/score_discrepancies.csv` is written by `compare_regrading_scores.py` and contains only large-difference students.

Columns:

- `student_id`
- `original_total`
- `review_total`
- `total_abs_diff`
- `total_pct_diff`
- `max_question_abs_diff`
- `changed_questions`
- `quantitative_flags`
- `difference_reason`
- `original_files`
- `copied_problem_files`
- `original_grading_json`
- `review_grading_json`

The first column must remain `student_id`.
